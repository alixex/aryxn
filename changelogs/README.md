# Changelogs

本目录记录项目开发过程中遇到的重要问题、bug 修复和解决方案。

## 文件命名规范

使用日期格式命名：`YYYY-MM-DD-简短描述.md`

例如：

- `2024-12-19-vite-sqlite-fixes.md`
- `2024-12-20-feature-update.md`

## 内容格式

每个 changelog 文件应包含：

1. **概述** - 本次更新的简要说明
2. **问题与解决方案** - 详细记录每个问题和解决方案
3. **配置变更总结** - 列出所有修改的文件
4. **测试建议** - 如何验证修复
5. **相关资源** - 相关文档链接

## 目的

- 📝 记录问题和解决方案，便于后续参考
- 🔍 帮助团队成员理解代码变更的原因
- 🐛 追踪 bug 修复历史
- 📚 积累项目知识库

## 查看最新更新

查看最新的 changelog：

```bash
ls -lt changelogs/ | head -5
```
